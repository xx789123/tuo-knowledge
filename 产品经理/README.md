## 核心技能

### PRD标准结构（2025版）
```
# 产品需求文档 (PRD)

## 1. 产品概述
### 1.1 背景与问题陈述
### 1.2 目标用户画像
### 1.3 成功指标 (KPIs)

## 2. 功能需求
### 2.1 功能列表 (Features)
### 2.2 用户故事 (User Stories)
### 2.3 流程图 (Flows)
### 2.4 页面清单 (Screens)

## 3. 非功能需求
### 3.1 性能要求
### 3.2 安全要求
### 3.3 兼容性要求
### 3.4 可用性要求

## 4. 交互设计
### 4.1 线框图链接
### 4.2 设计稿链接
### 4.3 动效说明

## 5. 验收标准
### 5.1 功能验收条件
### 5.2 测试用例

## 6. 风险与依赖
### 6.1 风险评估
### 6.2 外部依赖

## 7. 时间线
### 7.1 里程碑
### 7.2 发布计划
```

### 用户画像模板
```
姓名：[用户昵称]
年龄：XX岁
职业：[职业]
目标：
- [用户想要完成的主要目标]
痛点：
- [当前解决方案无法满足的需求]
使用场景：
- [何时、何地、如何使用产品]
```

### RICE评分计算
```
RICE = (Reach × Impact × Confidence) / Effort

- Reach（触达）：一个季度内多少用户受影响
- Impact（影响）：对用户指标的提升程度 (0.25/0.5/1/2/3)
- Confidence（信心）：对估算的自信程度 (50%/80%/100%)
- Effort（工作量）：人月数
```
